# CAP 定理

**CAP 定理**（也称 Brewer's Theorem）指出，分布式数据系统最多只能同时满足以下三个保证中的两个：

- **一致性（Consistency）**：每次读取都能收到最新的写入结果
- **可用性（Availability）**：每个请求都能收到（非错误）响应
- **分区容错性（Partition Tolerance）**：系统在网络分区的情况下仍能继续运行

> 在网络分区不可避免的分布式系统中，你必须在一致性和可用性之间做选择。

## 核心权衡

| 选择                  | 含义                           | 典型系统             |
| --------------------- | ------------------------------ | -------------------- |
| **CP**（一致 + 分区） | 分区时停止写入以保证一致性     | Zookeeper, etcd      |
| **AP**（可用 + 分区） | 分区时允许数据不一致但保持可用 | Cassandra, DynamoDB  |
| **CA**（一致 + 可用） | 不允许分区存在，等于单机系统   | 传统 RDBMS（单节点） |

## 关键理解

CAP 定理并不是让你在三种中选择两种，而是：**在发生网络分区时，你必须在 C 和 A 之间放弃一个。**

- 系统正常运行时（没有分区），你可以同时拥有 C 和 A
- 只有当分区发生时，才需要做取舍
